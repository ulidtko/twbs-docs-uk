Завантаження Bootstrap

Bootstrap має кілька простих способів "з чого починати", кожен з яких цікавий у залежності від рівня досвіду та конкретної потреби використання. Прогляньте що підходить найкраще саме вам.

Зібрані CSS, JS, та шрифти

Завантаження зібраних та мінімізованих версій наших CSS, JavaScript та шрифтів – хороший старт. Щоправда цей матеріал не містить жодної документації чи сирцевого коду.

Завантажте сирцевий код

Дане завантаження буде здійснено з GitHub, воно вміщує в собі свіжі версії сирцевого коду Bootstrap LESS та JavaScript.

Клонуйте або зробіть відгалуження через GitHub

Відвідайте нас на GitHub, щоб клонувати або зробити відгалуження проекту Bootstrap.

Встановлення через Bower

Щоб встановити та керувати стилями Bootstrap, JavaScript, та їх документацією використовуйте Bower.

$ bower install bootstrap

Bootstrap CDN

Люди з MaxCDN люб'язно надали підтримку CDN для Bootstrap CSS та JavaScript. Просто використовуйте ці лінки Bootstrap CDN.

<!-- Свіжа збірка мінімізованих CSS -->
<link rel="stylesheet" href="//netdna.bootstrapcdn.com/bootstrap/3.0.3/css/bootstrap.min.css">

<!-- Додаткові теми -->
<link rel="stylesheet" href="//netdna.bootstrapcdn.com/bootstrap/3.0.3/css/bootstrap-theme.min.css">

<!-- Свіжа збірка мінімізованої JavaScript -->
<script src="//netdna.bootstrapcdn.com/bootstrap/3.0.3/js/bootstrap.min.js"></script>

Компільовані Bootstrap LESS-файли

Якщо ви отримали сирцевий код Bootstrap, вам потрібно компілювати LESS-файли в ньому, щоб перетворити їх на робочі CSS-файли. Для компіляції ми використовуємо та офіційно підтримуємо лише Recess, який у Twitter є CSS помічником заснованим на less.js.

Що включено

Bootstrap доступний у двох формах, у яких ви знайдете логічно згруповані каталоги та файли, представлені в зібраному та мінімізованому варіанті.

Необхідність в jQuery

Будь-ласка майте на увазі, що всі JavaScript плагіни потребують включення jQuery, як це показано в стартовому шаблоні. У файлі bower.json можна побачити які версії jQuery підтримуються.

Збірка коду Bootstrap

Після завантаження, розархівуйте стиснені каталоги. Ви маєте побачити щось схоже на це:

bootstrap/
├── css/
│   ├── bootstrap.css
│   ├── bootstrap.min.css
│   ├── bootstrap-theme.css
│   └── bootstrap-theme.min.css
├── js/
│   ├── bootstrap.js
│   └── bootstrap.min.js
└── fonts/
    ├── glyphicons-halflings-regular.eot
    ├── glyphicons-halflings-regular.svg
    ├── glyphicons-halflings-regular.ttf
    └── glyphicons-halflings-regular.woff

Це є найбільш базова форма Bootstrap: зібрані файли, для їх швидкої вставки в будь-який веб-проект. Ми надаємо збірку CSS та JS (bootstrap.*), а також її мінімізований варіант (bootstrap.min.*). Як додаткові опції тем Bootstrap – також надаються шрифти з Glyphicons.

Сирцевий код Bootstrap

Завантаження сирцевого коду Bootstrap вміщує збірку CSS, JavaScript, та шрифтів, а також сирцевий код LESS, JavaScript, та документацію. Точніше, завантаження має такі елементи:

bootstrap/
├── less/
├── js/
├── fonts/
├── dist/
│   ├── css/
│   ├── js/
│   └── fonts/
├── docs-assets/
├── examples/
└── *.html

В каталоги less/, js/, та fonts/ вміщено сирцевий код для наших CSS, JS, та значків шрифта (відповідно). Каталог dist/ вміщує весь список з верхньої секції завантаження в прекомпільованому вигляді (готовому до використання, на відміну від сирцевого коду). Каталоги docs-assets/, examples/, та всі файли *.html призначені для нашої документації. Окрім цього, будь-який інший включений файл надає підтримку для пакетів, інформацію ліцензування, та розробку.

Базовий шаблон

Можна почати з цього базового шаблону, або з прикладів, наведених нижче. Ми сподіваємось, що ви будете адаптувати їх під власні потреби.

Нижче надано HTML-код, який використовує мінімізовану версію Bootstrap-документу.

<!DOCTYPE html>
<html>
  <head>
    <title>Bootstrap 101 Template</title>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <!-- Bootstrap -->
    <link href="css/bootstrap.min.css" rel="stylesheet">

    <!-- Вставка для підтримки браузером IE8 елементів HTML5 та медіа запитів -->
    <!-- ПОПЕРЕДЖЕННЯ: Respond.js не працює, якщо ви проглядаєте сторінку через file:// -->
    <!--[if lt IE 9]>
      <script src="https://oss.maxcdn.com/libs/html5shiv/3.7.0/html5shiv.js"></script>
      <script src="https://oss.maxcdn.com/libs/respond.js/1.3.0/respond.min.js"></script>
    <![endif]-->
  </head>
  <body>
    <h1>Привіт, світ!</h1>

    <!-- jQuery (потрібно для JavaScript плагінів Bootstrap) -->
    <script src="https://code.jquery.com/jquery.js"></script>
    <!-- Включення мінімізованої збірки всіх плагінів. -->
    <script src="js/bootstrap.min.js"></script>
  </body>
</html>

Блокування чутливості

Bootstrap автоматично адаптує перегляд сторінок під декілька розмірів екранів. Тут показано як заблокувати таку можливість, щоб сторінки відображались як в цьому нечутливому шаблоні.

Кроки для блокування чутливого перегляду

  1. Не створюйте <meta> з іменем viewport як про це сказано в документах CSS
  2. Скасуйте width в .container для кожного рівня сітки певної ширини, наприкла так width: 970px !important; Переконайтеся, що цей код йде після включення основного коду Bootstrap CSS. При бажанні можна уникнути !important з медіа-запитами або зі спеціальним селектором.
  3. Якщо ви використовуєте навігаційне меню, повидаляйте всі меню, які можуть згортатись/розгортатись.
  4. Для макету сітки використовуйте класи .col-xs-* як додаток або замість класів medium/large. Не хвилюйтесь, сітка з класом xs (extra-small) масштабується для всіх розмірів екранів.

Вам все ще потрібний Respond.js для IE8 (оскільки наші медіа-запити все ще існують і повинні оброблятись). Це заблокує властивість "мобільність сайту" Bootstrap.

Додаткові примітки

Інші зміни в v3.0 є не настільки очевидними. Базові класи, основні стилі та поведінки були скориговані для гнучкості та нашого підходу мобільність на першому місті. Ось неповний список:

  • За промовчанням, текстові елементи керування форми тепер отримують мінімальний стиль. Для керування кольором фокуса та закругленими кутами застосовуйте клас .form-control.
  • Текстові елементи керування формою з класом .form-control тепер за промовчанням мають ширину зі значенням 100%. Вкладайте форми вводу в середину <div class="col-*"></div> для управління їх шириною.
  • .badge більше немає контекстних класів (-success,-primary,і т.д.).
  • .btn повинен також використовувати .btn-default, щоб створити кнопку за промовчанням.
  • .container та .row тепер рідинні (базуючись на процентах).
  • Картинки віднині нечутливі за промовчанням. Використовуйте .img-responsive, щоб отримати рідинний розмір <img>.
  • Тепер значки .glyphicon базуються на шрифтах. Значкам також необхіді базові та спеціальні класи (наприклад .glyphicon .glyphicon-asterisk).
  • Typeahead було видалено, на користь Twitter Typeahead.
  • Модальна розмітка істотно змінилась. Тепер блоки із .modal-header, .modal-body, та .modal-footer обгортаються в .modal-content та .modal-dialog, для кращої мобільності стилю та поведінки.
  • Завантаження HTML через модальну опцію remote зараз включено в .modal (раніше включалось в .modal-body). Це дозволяє вам також легко варіювати модальні заголовки та нижній колонтитул, а не лише модальне тіло.
  • JavaScript події мають простори імен (namespaced). Наприклад, щоб обробити подію "show" в модалі, використовуйте 'show.bs.modal'. Для закладок "shown" використовуйте 'shown.bs.tab', і т.д.

Більш докладну інформацію стосовно апгрейду до v3.0 проглянте на Bootply, де учасники обговорень викладають також і частини коду.

Підтримка браузерами

Bootstrap спроектовано для найкращої роботи в нових браузерах, тобто старі браузери не завжди можуть правильно відображати стилі.

Підтримка браузерами

Зокрема, ми підтримуємо останні версії:

  • Chrome (Mac, Windows, iOS, та Android)
  • Safari (тільки для Mac та iOS, оскільки версія для Windows більше не підтримується)
  • Firefox (Mac, Windows)
  • Internet Explorer
  • Opera (Mac, Windows)

Bootstrap повинен досить добре виглядати та поводитись в Chromium та Chrome для Linux, Firefox для Linux, та Internet Explorer 7, хоча вони не офіційно підтримуються.

Internet Explorer 8 та 9

Internet Explorer 8 та 9 також підтримується, однак пам'ятайте, що деякі властивості CSS3 та елементи HTML5 не в повній мірі підтримуються даними браузерами. Крім того, Internet Explorer 8 потребує використання Respond.js, щоб дозволити підтримку медіа запитів.

Властивість Internet Explorer 8 Internet Explorer 9
border-radius Не підтримується Підтримується
box-shadow Не підтримується Підтримується
transform Не підтримується Підтримується з префіксом -ms
transition Не підтримується
placeholder Не підтримується

Відвідайте Чи можу я використовувати... для більш детального ознайомлення, які властивості CSS3 та HTML5 підтримуються різними браузерами.

Internet Explorer 8 та Respond.js

Візміть до уваги наступні застереження, при використанні Respond.js у вашому середовищі для розробки, чи в робочому середовищі для Internet Explorer 8.

Міждоменне зберігання Respond.js та CSS

Якщо ви зберігаєте Respond.js і CSS на різних (суб)доменах (наприклад, на CDN) – це вимагає деяких додаткових установок. Ознайомтесь з документацією Respond.js для уточнень.

Respond.js та file://

Через правила безпеки браузера, Respond.js не працює при перегляді через протокол file:// (наприклад коли ви відкриваєте HTML файл локально). Щоб протестувати налаштування чутливості в IE8, проглядайте ваші сторінки через HTTP(S). Ознайомтесь з документацією Respond.js для уточнень.

Respond.js та @import

Respond.js не працює з CSS, на які посилаються через @import. Зокрема, деякі конфігурації Drupal, як відомо, використовують @import. Ознайомтесь з документацією Respond.js для уточнень.

Internet Explorer 8 та box-sizing

IE8 не в повній мірі підтримує box-sizing: border-box; при комбінуванні з min-width, max-width, min-height, або max-height. З цієї причини, починаючи з v3.0.1, ми більше не підтримуємо max-width в .container.

Режим сумісності з IE

Bootstrap не підтримує режиму сумісності в старих версіях Internet Explorer. Щоб бути впевненим, що ви використовуєте останню версію такого режиму для IE, прогляньте чи включений відповідний тег <meta> на вашій сторінці:

<meta http-equiv="X-UA-Compatible" content="IE=edge">

Цей тег включено у всі сторінки документації та прикладів для забезпечення найкраще-можливого відображення в кожній з версій Internet Explorer.

Прогляньте ці питання на StackOverflow для уточнень.

Internet Explorer 10 на Windows 8 та Windows Phone 8

Internet Explorer 10 не відрізняє ширину пристрою від ширини вікна перегляду, і таким чином не правильно застосовує медіазапити в Bootstrap CSS. Як правило, це можна швидко полагодити додавши наступний код CSS:

@-ms-viewport       { width: device-width; }

Однак це не працює на пристроях з Windows Phone 8, де відображається загальний вигляд робочого столу, замість вузької області перегляду "для телефону". Щоб вирішити дану проблему вам необхідно включити наступний код CSS та JavaScript, допоки Microsoft не полагодить це.

@-webkit-viewport   { width: device-width; }
@-moz-viewport      { width: device-width; }
@-ms-viewport       { width: device-width; }
@-o-viewport        { width: device-width; }
@viewport           { width: device-width; }

if (navigator.userAgent.match(/IEMobile\/10\.0/)) {
  var msViewportStyle = document.createElement("style")
  msViewportStyle.appendChild(
    document.createTextNode(
      "@-ms-viewport{width:auto!important}"
    )
  )
  document.getElementsByTagName("head")[0].appendChild(msViewportStyle)
}

Для отримання додаткової інформації та керівництва з використання, прогляньте Windows Phone 8 та Device-Width.

Щоб привернути увагу, ми включили це в якості прикладів у документації Bootstrap.

Процентне округлення в Safari

У версіях Safari v6.1 для OS X та Safari для iOS v7.0.1, при використанні класу сітки .col-*-1, рушій Safari має деякі проблеми з кількістю десяткових знаків. Тобто якщо ви маєте 12 окремих колонок сітки, ви помітете, що вони будуть коротшими в порівнянні з іншими рядами колонок. Тут малощо можна змінити (прогляньте #9282), але у вас є кілька варіантів:

  • Додати .pull-right до вашої останньої колонки в сітці, щоб отримати жорстке вирівнювання по правому краю.
  • Змінити проценти вручну, щоб отримати ідеальне округлення для Safari (більш складно, ніж у першому варіанті)

Ми будемо слідкувати за цією проблемою, і якщо буде більш просте рішення, то оновимо наш код.

Модальні вікна та мобільні пристрої

Переповнення та прокрутка

Підтримка overflow: hidden для елементу <body> досить обмежена в iOS та Android. У браузері цих пристроїв, при прокруткі за межі верхньої або нижньої модальної частини, починає прокручуватись вміст в <body>.

Віртуальна клавіатура

Також зауважте – якщо ви використовуєте форму вводу в модальному вікні – iOS має помилку відображення, через яку не оновлюється позиція фіксованих елементів при виклику віртуальної клавіатури. Існують деякі обхідні шляхи для цього, у тому числі – трансформують елементи через position: absolute, або при виявленні фокусу на елементі викликають таймер і пробують виправити це вручну. Bootstrap не обробляє такі помилки, отже вам потрібно самим це вирішити.

Браузерне масштабування

При зміні розміру сторінки, неминуче виявляється некоректне відображення деяких елементів: це стосується і Bootstrap, і решти веб-документу. У залежності від конкретного випадку, ми можемо виправити це (при потребі ви можете відкрити відповідний запит, але спочатку необхідно пошукати чи немає аналогічних запитів). Тим не менш, але як правило ми ігноруємо їх, оскільки вони часто не мають прямого вирішення, окрім створення обхідних шляхів у вигляді хаків (hacky).

Підтримка третіх сторін

Хоча ми офіційно не підтримуємо сторонні плагіни або доповнення, ми пропонуємо кілька корисних порад, які допоможуть уникнути можливих проблем у ваших проектах.

Розмірність блоків

Деяке стороннє програмне забезпечення, включно з Google Maps та Google Custom Search Engine, конфліктують з Bootstrap через * { box-sizing: border-box; }, тобто правило, котре створює padding (відступи) в підсумку не впливає на ширину елемента. Дізнайтесь більше про це: box model and sizing at CSS Tricks.

В залежності від контексту, ви можете перевизначити box-sizing лише там, де це необхідно (Варіант 1), або скинути його для цілих областей (Варіант 2).

/* Скидання box-sizing
 *
 * Скидання на окремому елементі чи перевизначення області для уникнення конфлікту через
 * глобальне налаштування блочної моделі Bootstrap. Два варіанти: індивідуальне перевизначення та
 * скидання – є доступними у вигляді простого CSS та некомпільованого LESS формату.
 */

/* Варіант 1A: Перевизначення окремого елемента блочної моделі через CSS */
.element {
  -webkit-box-sizing: content-box;
     -moz-box-sizing: content-box;
          box-sizing: content-box;
}

/* Варіант 1B: Перевизначення окремого елемента блочної моделі використовуючи Bootstrap LESS mixin */
.element {
  .box-sizing(content-box);
}

/* Варіант 2A: Скидання цілої області за допомогою CSS */
.reset-box-sizing,
.reset-box-sizing *,
.reset-box-sizing *:before,
.reset-box-sizing *:after {
  -webkit-box-sizing: content-box;
     -moz-box-sizing: content-box;
          box-sizing: content-box;
}

/* Варіант 2B: Скидання цілої області за допомогою налаштування LESS mixin */
.reset-box-sizing {
  &,
  *,
  *:before,
  *:after {
    .box-sizing(content-box);
  }
}
.element {
  .reset-box-sizing();
}

Доступність

Bootstrap слідує загальним веб-стандартам, та при мінімальних додаткових зусиллях, може бути використаний для створення сайтів, доступних через AT.

Пропуск навігації

Якщо ваше навігаційне меню має багато лінків та знаходиться перед головним вмістом в DOM, додайте лінк Перейти до головного вмісту безпосередньо після тегу <body>. (прочитайте чому)

<body>
  <a href="#content" class="sr-only">Перейти до головного вмісту</a>
  <div class="container" id="content">
    Головний вміст сторінки.
  </div>
</body>

Вкладені заголовки

При вкладанні заголовків (<h1> – <h6>), ваш основний заголовок документа повинен бути <h1>. Наступні заголовки повинні логічно вибудовуватись використовуючи <h2> – <h6>, щоб програма, яка зчитує заголовки, могла сконструювати зміст сторінки.

Дізнатесь більше про HTML CodeSniffer та Penn State's AccessAbility.

Ліцензія FAQs

Bootstrap випущено під ліцензією Apache 2 та має авторське право Twitter 2013. Зведена до менших частин, вона може бути описана наступними умовами.

Вона дозволяє:

  • Вільно завантажувати та використовувати Bootstrap цілком або частинами, для особистих, корпоративних чи комерційних цілей
  • Використовувати Bootstrap в пакетах або в дистрибуції, яку ви самі створюєте

Вона забороняє:

  • Поширювати будь-яку частину Bootstrap без посилання на джерело
  • Використовувати будь-які позначки, які належать Twitter, щоб у такий спосіб явно чи неявно продемонструвати начебто Twitter схвалює ваш дистрибутив
  • Використовувати будь-які позначки, які належать Twitter, щоб у такий спосіб явно чи неявно продемонструвати начебто ви створюєте програмне забезпечення за запитом Twitter

Вона вимагає:

  • Включення копії даної ліцензії в будь-який дистрибутив, який вміщує Bootstrap
  • Дати чітке привласнення Twitter'у в будь-якому дистрибутиві, що вміщує Bootstrap

Вона не вимагає:

  • Включати сирцевий код Bootstrap itself, or of any modifications you may have made to it, in any redistribution you may assemble that includes it
  • Надавати зміни, що ви внесли до Bootstrap для демонстрації в проекті Bootstrap (хоча зворотній зв'язок заохочується)

Повна версія ліцензії Bootstrap розміщується в проекті репозиторію.

Компонування Bootstrap

Bootstrap найкраще обслуговувати, коли в середовищі розробки розділяти кожну залежність від нього на окремі версії. Дотримуючись цього правила – у майбутньому ваш апгрейд до нової версії Bootstrap буде легким.

Після того як ви завантажили та підключили стилі і скрипти Bootstrap, ви можете налаштовувати їх компоненти. Щоб це зробити, просто створіть нову таблицю стилів (LESS, якщо бажаєте, або просто CSS).

Просту збірку, чи мінімізовану?

Якщо ви не плануєте читати CSS, вибирайте їх мінімізовану версію. Це той же код, просто компактніший. Мінімізовані таблиці стилів використовують меншу ширину каналу, що є добре, особливо в робочому (production) середовищі.

І надалі, включайте будь-які компоненти Bootstrap та HTML вміст, для створення шаблонів сторінок вашого сайту.

Компонування складових

Ви можете налаштовувати компоненти в різній мірі, але в більшості виокремлюються дві категорії: легке налаштування та повна перебудова. Є багато різних прикладів від третіх сторін, які дозволяють реалізувати обидві категорії.

Ми визначаємо легке налаштування як поверхневі зміни, наприклад, зміни кольору та шрифту для існуючих компонентів Bootstrap. Прикладом легкого налаштування є Twitter Translation Center (написаний @mdo). Давайте подивимось як реалізується налаштування кнопки .btn-ttc на цьому сайті.

Для створення основи кнопки Bootstrap необхідно лише один клас .btn. Тепер ми розширюватимемо стиль .btn новим модифікованим класом .btn-ttc, який пізніше створимо. Це дозволяє змінювати вигляд при мінімальному зусиллі.

Наша кнопка, яку ми налаштовуємо, буде мати приблизно такий код:

<button type="button" class="btn btn-ttc">Зберегти зміни</button>

Зауважте як .btn-ttc додається до стандартного класу .btn.

Щоб завершити налаштування, додайте наступний користувальницький код стилів CSS:

/* Користувальницька кнопка
-------------------------------------------------- */

/* Перевизначення базового стилю .btn */
/* Встановлення текстових та фонових змін для трьох основних станів: default, hover, та active (при кліку). */
.btn-ttc,
.btn-ttc:hover,
.btn-ttc:active {
  color: white;
  text-shadow: 0 -1px 0 rgba(0, 0, 0, 0.25);
  background-color: #007da7;
}

/* Встановлення користувальницького градієнту кольору */
/* Зауважте: вам необхідно буде включати всі відповідні градієнти для різних браузерів та стандартів. */
.btn-ttc {
  background-repeat: repeat-x;
  background-image: linear-gradient(top, #009ED2 0%, #007DA7 100%);
  ...
}

/* Оголошення hover */
/* Легке оголошення hover – просте бідняття градієнту на незначну величину. Додайте інші коригування, якщо необхідно. */
.btn-ttc:hover {
  background-position: 0 -15px;
}

Коротше кажучи: Проглянте джерело стилю та продублюйте селектори для модифікації.

В підсумку, базовий робочий процес буде мати такі пункти:

  • Знайдіть код у збірці CSS Bootstrap для кожного елементу, котрий хочете налаштувати.
  • Скопіюйте компоненти селектора та стилі і вставте їх до ваших стилів, які налаштовуєте. Наприклад, щоб налаштувати фон навігаційного меню, просто скопіюйте опис стилю .navbar.
  • У ваших стилях, які налаштовуєте, редагуйте CSS, які ви щойно скопіювали з Bootstrap. Не потрібно дописувати додаткові класи, чи додавати в кінці !important. Просто збережіть їх.
  • Прогляньте що у вас вийшло і при потребі повторіть попередні кроки.

Як тільки ви навчитеся робити легке налаштування, візуальна повна перебудова буде такою ж простою. Наприклад для сайту Karma, котрий використовує Bootstrap зі скиданням CSS та важкими модифікаціями, потрібно добре попрацювати. Але тут використовується той же принцип: включаються спочатку стилі Bootstrap за промовчанням, потім застосовується їх налаштування.

Альтернативні методи налаштування

Хоча не рекомендується для користувачів, погано знайомих з Bootstrap, все ж ви можете використовувати один з двох альтернативних методів для налаштування. Перший полягає в зміні коду файлу .less (супер-ускладнює апгрейд), другий – створення мапінгу між LESS кодом та вашими власними класами через mixins. На даний момент, жоден з цих методів тут покищо не документовано.

Видалення непотрібного коду

Не потрібно для кожного сайту все робити за допомогою Bootstrap, особливо у виробничому (production) середовищі, де потрібно оптимізувати пропускну спроможність. Ми рекомендуємо повидаляти все зайве використовуючи наш Компонувальник.

Для цього просто познімайте галочки з усіх компонентів, функцій, або груп компонентів, які вам не потрібні. Завантажуйте нові змінені файли та замінюйте ними файли Bootstrap за промовчанням. Ви отримаєте свіжі файли Bootstrap, але без функцій, котрі ви порахували зайвими. Кожна така компонована збірка містить компільовані та мінімізовані версії.
