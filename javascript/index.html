Огляд

Індивідуально чи збіркою

Плагіни можуть підключатись окремо (використовуючи окремі файли Bootstrap *.js), або всі за один раз (використовуючи bootstrap.js або мінімізовані bootstrap.min.js).

Немає потреби підключати обидва файли.

Повний набір плагінів міститься як у bootstrap.js, так і у bootstrap.min.js.

Залежність плагіну

Деякі плагіни і CSS та їх компоненти мають залежність від інших плагінів. Якщо ви включаєте плагіни окремо, прогляньте їх документацію, щоб переконатись, що ви врахували їх залежність. Також зауважте, що всі плагіни залежать від jQuery (це означає, що jQuery повинен підключатись перед файлами плагінів). Прогляньте наш файл bower.json, щоб дізнатись яка версія jQuery підтримується.

Атрибути даних

Ви можете використовувати всі Bootstrap плагіни виключно на основі розмітки API, без необхідності писати код на JavaScript. Це є першочерговий API клас для Bootstrap та повинен розглядатись в першу чергу при використанні плагінів.

Тим не менш, у деяких ситуаціях може бути бажаним відключення цієї функціональності. Таким чином, ми також надаємо можливість відключити API-атрибути, через відв'язку всіх подій у просторі імен (namespaced) документа, за допомогою data-api. Це виглядає приблизно так:

$(document).off('.data-api')

Також ви можете націлитись на певний плагін, просто включивши ім'я плагіну поруч з простором імен data-api, як у прикладі:

$(document).off('.alert.data-api')

Програмний API

Ми також вважаємо, що ви повинні мати можливість використовувати всі плагіни Bootstrap чисто на JavaScript API. Всі публічні API є автономними, але їхні методи можуть послідовно зв'язуватись та повертати готовий набір для застосування.

$('.btn.danger').button('toggle').addClass('fat')

Всі методи повинні приймати в якості необов'язкового параметра або об'єкт, або рядок, котрий націлений на конкретний метод, або нічого (і таким чином просто ініціалізувати плагін з поведінкою за промовчанням):

$('#myModal').modal()                      // ініціалізується із значенням за промовчанням
$('#myModal').modal({ keyboard: false })   // ініціалізується без клавіатури
$('#myModal').modal('show')                // ініціалізується та викликає негайне відображення

Кожен плагін також публікує свій власний конструктор у властивості Constructor: $.fn.popover.Constructor. Якщо ви хочете отримати конкретний екземпляр плагіна, завантажуйте його безпосередньо з елемента: $('[rel=popover]').data('popover').

Без конфлікту

Інколи необхідно використовувати плагіни Bootstrap з іншими UI фрейморками. У такому разі інколи можуть перетинатись їхні простори імен. Якщо таке трапляється, ви можете викликати .noConflict в тому плагіні, котрий має повернути попереднє значення.

var bootstrapButton = $.fn.button.noConflict() // повертає $.fn.button до попередньо встановленого значення
$.fn.bootstrapBtn = bootstrapButton            // призначає $().bootstrapBtn функціональність Bootstrap

Події

Bootstrap надає користувальницькі події для більшості унікальних дій плагіна. Як правило, вони йдуть у формі інфінітива чи дієприкметника – причому інфінітив (наприклад, show) спрацьовує на початку події, а форма дієприкметника (наприклад, shown) спрацьовує по завершенню дії.

Станом на 3.0.0, всі події Bootstrap з іменованими просторами.

Всі події в інфінітиві надають preventDefault функціональність. Це дає можливість зупиняти виконання дій перед їх стартом.

$('#myModal').on('show.bs.modal', function (e) {
  if (!data) return e.preventDefault() // зупиняє модаль перед її показом
})

Бібліотеки третіх сторін

Bootstrap неофіційно підтримує бібліотеки JavaScript третіх сторін, такі як Prototype або jQuery UI. Незважаючи на те, що ми надаємо .noConflict та події з іменованими просторами, все одно можуть бути проблеми сумісності, котрі вам доведеться виправляти власноруч. Задайте питання через список розсилки, якщо вам потрібна допомога.

Переходи transition.js

Про переходи

Для простих ефектів переходу, включіть transition.js поруч з іншими файлами JS. Але якщо ви використовуєте загальну збірку bootstrap.js (чи її мінімізований варіант), цей файл немає необхідності підключати – його код вже включено.

Що всередені

Transition.js є базовим помічником для подій transitionEnd, та для емулятора переходів CSS. Він використовується іншими плагінами для перевірки підтримки переходів CSS та для відловлення висячих переходів.

Приклади

Зробіть модалі доступними

Переконайтесь, що додали role="dialog" до .modal, атрибут aria-labelledby="myModalLabel" для посилання на назву модалі, та aria-hidden="true" для сповіщення assistive technologies (допоміжних технологій), щоб пропускали DOM-елементи модальних вікон.

Крім цього, ви можете надати опис діалогової модалі за допомогою aria-describedby в .modal.

Використання

Плагін для модалі, при запиті, переключає видимість вашого прихованого вмісту через атрибути даних чи JavaScript. Він також додає .model-open до <body> для перевизначення поведінки за промовчанням, стосовно прокручування сторінок, та генерує .modal-backdrop, щоб надати область за межами модалі, при клікі по якій, скасовується показ модалі.

Через атрибути даних

Активуйте модальні вікна без написання JavaScript. Встановіть data-toggle="modal" в контролері елемента, такого як кнопка, поруч з data-target="#foo" або href="#foo", щоб націлити визначену модаль для переключення.

<button type="button" data-toggle="modal" data-target="#myModal">Запустити модаль</button>

Через JavaScript

Викликайте модаль за допомогою id myModal з лаконічним записом на JavaScript:

$('#myModal').modal(параметри)

Параметри

Параметри можуть передаватись через атрибути даних чи JavaScript. У випадку передачі через атрибути даних, додайте приставку з назвою параметра до data-, як у прикладі: data-backdrop="".

Назва тип за промовчанням опис
backdrop логічний чи рядковий 'static' true Включає елемент modal-backdrop. Можна також вказати static для backdrop, котрий не закривається при натисканні на модаль.
keyboard логічний true Зкриває модаль при клікі на клавіші escape.
show логічний true Показує модаль при ініціалізації.
remote шлях false

Якщо надано віддалений URL, вміст буде завантажуватись через jQuery-метод load та вставлятись в кореневий елемент модалі. Якщо ви використовуєте data-api, ви можете ще використовувати атрибут href для визначення віддаленого ресурсу. Приклад такого варіанту показаний нижче:

<a data-toggle="modal" href="remote.html" data-target="#modal">Клікніть мене</a>

Методи

.modal(параметри)

Активізує вміст як модальний. Приймає необов'язковий параметр з типом даних об'єкт.

$('#myModal').modal({
  keyboard: false
})

.modal('toggle')

Ручне переключення модалі. Повертає до ініціатора перед фактичним показом чи приховуванням модалі (тобто, перед запуском подій shown.bs.modal або hidden.bs.modal).

$('#myModal').modal('toggle')

.modal('show')

Ручне відображення модалі. Повертає до ініціатора перед фактичним показом модалі (тобто, перед запуском подій shown.bs.modal).

$('#myModal').modal('show')

.modal('hide')

Ручне приховування модалі. Повертає до ініціатора перед фактичним приховуванням модалі (тобто, перед запуском подій hidden.bs.modal).

$('#myModal').modal('hide')

Відслідковування прокрутки scrollspy.js

Використання

Через атрибути даних

Щоб легко додати поведінку Відслідковувача Прокрутки (ScrollSpy) до вашої верхньої навігаційної панелі, додайте data-spy="scroll" до елемента, за яким ви хочете слідкувати (найбільш типово – це буде <body>). Потім додайте атрибут data-target з ID або класом батьківського елементу, будь-якого компоненту Bootstrap з класом .nav.

<body data-spy="scroll" data-target=".navbar-example">
  ...
  <div class="navbar-example">
    <ul class="nav nav-tabs">
      ...
    </ul>
  </div>
  ...
</body>

Через JavaScript

Викликайте Відслідковувача Прокрутки за допомогою JavaScript:

$('body').scrollspy({ target: '.navbar-example' })

Необхідна відповідність між ID та його лінком

Навігаційна панель повинна мати відповідність між ID та його лінком. Наприклад, <a href="#home">home</a> повинен мати відповідність в DOM із <div id="home"></div>.

Методи

.scrollspy('refresh')

При використанні Відслідковувача Прокрутки в поєднанні з додаванням або видаленням елементів з DOM, вам потрібно викликати оновлюючий метод, як в прикладі:

$('[data-spy="scroll"]').each(function () {
  var $spy = $(this).scrollspy('refresh')
})

Параметри

Параметри можуть передаватись через атрибути даних або JavaScript. Для атрибутів даних, додайте приставку з назвою параметра до data-, як у прикладі: data-offset="".

Назва тип за промовчанням опис
offset числовий 10 Пікселі для зміщення від верху, при обчисленні позиції прокрутки.

Розширення навігаційних вкладок

Цей плагін розширює компонент навігаційних вкладок, додаючи область переходів.

Використання

Включення області переходів вкладок через JavaScript (кожна вкладка потребує індивідуальної активації):

$('#myTab a').click(function (e) {
  e.preventDefault()
  $(this).tab('show')
})

Ви можете індивідуально активувати вкладки в декілька способів:

$('#myTab a[href="#profile"]').tab('show') // Виберіть вкладку по імені
$('#myTab a:first').tab('show') // Виберіть першу вкладку
$('#myTab a:last').tab('show') // Виберіть останню вкладку
$('#myTab li:eq(2) a').tab('show') // Виберіть третю вкладку (0-індексовано)

Розмітка

Ви можете активувати вкладку чи навігаційну кнопку і без написання коду JavaScript, просто вказавши в елементі data-toggle="tab" або data-toggle="pill". Додавання класів nav та nav-tabs до вкладки ul буде впроваджуватись стиль вкладок Bootstrap, при додаванні класів nav та nav-pills буде впроваджуватись стиль навігаційних кнопок.

<!-- Навігаційні вкладки -->
<ul class="nav nav-tabs">
  <li><a href="#home" data-toggle="tab">Головна</a></li>
  <li><a href="#profile" data-toggle="tab">Профіль</a></li>
  <li><a href="#messages" data-toggle="tab">Повідомлення</a></li>
  <li><a href="#settings" data-toggle="tab">Налаштування</a></li>
</ul>

<!-- Навігаційні панелі -->
<div class="tab-content">
  <div class="tab-pane active" id="home">...</div>
  <div class="tab-pane" id="profile">...</div>
  <div class="tab-pane" id="messages">...</div>
  <div class="tab-pane" id="settings">...</div>
</div>

Ефект згасання

Щоб створити ефект плавного відображення вкладок, додайте .fade до кожного .tab-pane. Перша панель вкладки повинна також мати .in для правильного плавного відображення вмісту.

Чотири напрямки

Неавтоматична функціональність

Для покращення продуктивності, дата-інтерфейси Підказок та Інформерів (Tooltip and Popover) неавтоматичні, тобто ви повинні ініціалізувати їх власноруч.

Підказки в групах кнопок та групах форм введення потребують спеціальних налаштувань

При використанні підказок всередені елементів .btn-group або .input-group, ви повинні визначити параметр container: 'body' (документація нижче), щоб уникнути небажаних побічних ефектів (наприклад, коли елемент стає ширшим та/або коли втрачається закругленння кутів, при виклику підказок чи інформерів).

Підказки на заблокованих елементах потребують обгортання

Щоб додати підказку до елемента disabled чи .disabled, вставте елемент у середену <div> та застосуйте підказку саме до цього <div>.

Використання

Плагін підказок генерує вміст та розмітку по запиту, і за промовчанням розміщує підказки після елементів, що їх викликали.

Виклик підказок через JavaScript:

$('#example').tooltip(параметри)

Розмітка

Згенерована розмітка підказок, досить проста, хоча і вимагає позиціонування (за промовчанням, встановлено top через плагін).

1 <div class="tooltip">
2   <div class="tooltip-inner">
3     Підказка!
4   </div>
5   <div class="tooltip-arrow"></div>
6 </div>

Параметри

Параметри можуть передаватись через атрибути даних або JavaScript. Для атрибутів даних, додайте приставку з назвою атрибута до data-, як у прикладі: data-animation="".

Назва тип за промовчанням опис
animation логічний true застосовує CSS для плавного переходу до підказки
html логічний false Вставляє HTML в підказку. При значенні false, буде використовуватись метод jQuery text для вставки вмісту в DOM. Використовуйте текст, якщо побоютесь XSS атаки.
placement рядковий | функція 'top' як позиціонувати підказку – top | bottom | left | right | auto.
При вказуванні "auto", підказка буде динамічно переорієнтовуватись. Нариклад, якщо розташування зазначено як "auto left", підказка буде відображуватись ліворуч, якщо це можливо, і праворуч – у противному разі.
selector рядковий false Якщо селектор надано, об'єкт підказки буде делеговано для зазначеної цілі.
title рядковий | функція '' назва за промовчанням, якщо атрибут title не зазначено
trigger рядковий 'hover focus' як викликати підказку – click | hover | focus | manual. Ви можете передавати декілька цих параметрів, розділених пробілами.
delay числовий | об'єкт 0

затримка показу та приховування підказок (ms) – не застосовується для ручного типу виклику

Якщо вказується число, затримка застосовується і для приховування, і для відображення

Структура об'єкту така: delay: { show: 500, hide: 100 }
container рядковий | false false

Зазначається елемент, до якого буде додану підказку. Наприклад: container: 'body'

html логічний false Вставляє HTML в інформер. При значенні false, буде використовуватись метод jQuery text для вставки вмісту в DOM. Використовуйте текст, якщо побоютесь XSS атаки.
placement рядковий | функція 'right' як позиціонувати інформер – top | bottom | left | right | auto.
При вказуванні "auto", інформер буде динамічно переорієнтовуватись. Нариклад, якщо розташування зазначено як "auto left", інформер буде відображатись ліворуч, якщо це можливо, і праворуч – у противному разі.
selector рядковий false якщо селектор надано, об'єкт підказки буде делеговано для зазначеної цілі. На практиці це використовується для додавання інформерів при включенні динамічного вмісту HTML. Прогляньте ось це та інформативний приклад.
trigger рядковий 'click' як інформери будуть викликатись – click | hover | focus | manual
title рядковий | функція '' значення назви за промовчанням, якщо атрибут title не вказано
content рядковий | функція '' значення вмісту за промовчанням, якщо атрибут data-content не вказано
delay числовий | об'єкт 0

затримка показу та відображення інформерів (ms) – не застосовується для ручного типу виклику

Якщо зазначено число, затримка застосовуватиметься і до приховування, і до відображення

Об'єкт має таку структуру: delay: { show: 500, hide: 100 }
container рядковий | false false

Зазначається елемент, до якого буде додано інформер. Наприклад: container: 'body'. Цей параметр особливо корисний тим, що дозволяє позиціонувати інформер у потоці документа біля викликаючого елемента - тим самим запобігаючи вспливанню біля викликаючого елемента, під час зміни розміру вікна.

Плагін згортання використовує декілька класів для забезпечення плавного піднімання:

  • .collapse приховування вмісту
  • .collapse.in відображення вмісту
  • .collapsing додається на початку переходу, та видаляється після його завершення

Ці класи можна знайти в component-animations.less.

Через атрибути даних

Просто додайте data-toggle="collapse" та data-target до елемента, для автоматичного встановлення контролю над елементами згортання. Атрибут data-target приймає селектор CSS для застосування згортання. Переконайтесь, що додали клас collapse до елементів для згортання. Якщо ви хочете, щоб за промовчанням елемент був відкритим, додайте ще клас in.

Щоб до групи елементів, які можуть згортатись, додати управління, схоже на акордеон, додайте атрибут даних data-parent="#selector". Прогляньте демонстрацію (вище), щоб побачити це в дії.

Анімація переходів не підтримується в Internet Explorer 8 та 9

Bootstrap для цієї анімації використовує виключно CSS3, але Internet Explorer 8 та 9 не підтримують необхідних властивостей CSS. Таким чином, у цих браузерах немає плавних переходів анімації. Ми свідомо вирішили не включати базованих на jQuery fallbacks для переходів.

Додавати підпис до ваших слайдів стає легкою справою завдяки елементу .carousel-caption всередені будь-яких .item. Це місце майже для будь-якого додаткового HTML, причому вміст буде автоматично вирівняно та форматовано.

Ярлик третього слайду

Тут довільний текст. Praesent commodo cursus magna, vel scelerisque nisl consectetur.

  <img src="..." alt="...">
  <div class="carousel-caption">
    <h3>...</h3>
    <p>...</p>
  </div>
</div>

Проблема доступності

Компонент каруселі в основному не сумісний зі стандартами доступності. Якщо вам потрібна сумісність, розгляньте будь-ласка інші параметри для презентації вашого вмісту.

Використання

Через атрибути даних

Використовуйте атрибути даних, щоб легко контролювати позиціонування каруселі. data-slide приймає ключові слова prev або next, котрі змінюють позицію слайду відносно його поточної позиції. Також ви можете використовувати data-slide-to для передачі вихідного індексу слайду для каруселі data-slide-to="2", котрий зміщує позицію слайду до певного індексу починаючи від 0.

Атрибут data-ride="carousel" використовується для позначення каруселі як анімації, на початку завантаження сторінки.

Через JavaScript

Викликайте карусель вручну із:

Параметри можуть передаватись через атрибути даних або JavaScript. Для атрибутів даних, додайте приставку з назвою параметра до data-, як у прикладі: data-interval="".

Назва тип за промовчанням опис
interval числовий 5000 Кількість мілісекунд для затримки між автоматичним зацикленням елементів. Якщо в якості параметра передано false, карусель не буде автоматично зациклюватись.
pause рядковий "hover" Пауза в прокручуванні каруселі при наведенні курсору (mouseenter) та продовження прокрутки, коли курсор миші забирають (mouseleave) з каруселі.
wrap логічний true Чи повинна карусель зациклено прокручуватись, чи повинна жорстко зупинитись.

Методи

.carousel(параметри)

Ініціалізує карусель з необов'язковим параметром типу об'єкт, та починає цикл прокручувань елементів.

$('.carousel').carousel({
  interval: 2000
})

.carousel('cycle')

Прокручування елементів каруселі зліва на право.

.carousel('pause')

Припинення прокручування елементів каруселі.

.carousel(число)

Прокручування каруселі до певного кадру (починаючи з 0, подібно до масиву).

.carousel('prev')

Перейти до попереднього елемента.

.carousel('next')

Перейти до наступного елемента.

Приклад

Субнавігація ліворуч є живим прикладом роботи плагіну affix.

Використання

Використовуйте плагін affix за допомогою атрибутів даних, або напишіть свій власний JavaScript. В обох випадках ви повинні надати CSS для позиціонування вмісту.

Позиціонування через CSS

Плагін affix переключається між трьома класами, кожен з яких відповідає певному стану: .affix, .affix-top, та .affix-bottom. Ви повинні власноруч створити стилі для цих класів (незалежно від цього плагіну), для обробки поточної позиції.

Ось як працює плагін affix:

  1. Спочатку, плагін додає .affix-top для позначення елемента, що знаходиться найвище. На даний момент позиціонування через CSS не потрібне.
  2. Прокручування повз елемент, котрий ви хочете закріпити, повинно актуалізувати закріплення. Це той момент, коли .affix замінює .affix-top та встановлює position: fixed; (забезпечене кодом Bootstrap CSS).
  3. Якщо нижнє зміщення визначене, прокручування вниз має замінити .affix на .affix-bottom. Оскільки зміщення є необов'язковим, вам необхідно встановити відповідні CSS. У даному випадку, додайте position: absolute;, коли це необхідно. Плагін використовує атрибути даних або параметри JavaScript для визначення позиції елемента.

Дотримуйтесь вищезазначених кроків, щоб надалі встановити CSS для будь-якого використання опцій.

Через атрибути даних

Можна легко надати поведінку affix будь-якому елементу, просто додавши data-spy="affix" до елементу, за яким ви хочете слідкувати. Використовуйте зміщення для визначення, коли слід вимикати плагін для прикріпленого елемента.

<div data-spy="affix" data-offset-top="60" data-offset-bottom="200">
  ...
</div>

Через JavaScript

Виклик плагіну affix через JavaScript:

  $('#myAffix').affix({
    offset: {
      top: 100
    , bottom: function () {
        return (this.bottom = $('.bs-footer').outerHeight(true))
      }
    }
  })

Параметри

Параметри можуть передаватись через атрибути даних або JavaScript. Для атрибутів даних, додайте приставку з назвою параметра до data-, як у прикладі: data-offset-top="200".

Назва тип за промовчанням опис
offset числовий | функція | об'єкт 10 Пікселі для зсуву екрана, при обрахунку позиції прокрутки. Якщо надано одиничне число, зсув буде застосовуватись і вгорі, і внизу. Щоб надати унікальності зсуву знизу та згори, просто надайте такий об'єкт offset: { top: 10 } чи такий offset: { top: 10, bottom: 5 }. Використовуйте функцію, коли вам потрібно динамічно обраховувати зсув.
